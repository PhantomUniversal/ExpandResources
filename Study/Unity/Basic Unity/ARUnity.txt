[Vuforia 마커와 인식률]   
● 마커  
  ▶ Single Image : 평면 이미지를 이용한 가장 일반적인 마커 유형
  ▶ Cuboid : 큐브 형태의 마커로 정사각형 또는 직사각형의 입방체를 마커로 인식하는 유형
  ▶ Cylinder : 원통형 또는 컴과 같은 형태를 마커로 인식하는 유형
  ▶ 3D Object : 3D 객체를 별도의 오브젝트 스캐너 앱을 이용해 스캔한 후 해당 객체를 마커로 인식하는 유형

● 인식률
  ▶ 불규칙한 패턴의 무늬 필요
  ▶ 명암 대비(Contrast)가 높은 이미지가 필요
     ▶ 이미지 명암 대비를 만들기 위해서는 Open Cv가 필요
        ▶ 특정 영역의 명암 대비를 구분하기 위해서 편집할려고 사용 
        ▶ 사물 이미지를 특정 부분을 인식 하기 위해 사용
           ▶ Python과 Yolo를 이용해 사물인식률을 높이자 
  ▶ Vuforia인식률은 별점으로 구분(1~5개 존재) // 별 4개이상의 이미지를 만들자
  ▶ Vuforia가 이미지를 분석할 때 노란색 십자가 모양 아이콘이 발생
     ▶ 십자가 모양 아이콘을 Feature라고 한다. 
        ▶ Feature의 개수가 많을 수록 별점이 높아진다.(인식률 결정)

[Vuforia Object]
● ImageTarget
  ▶ GameObject → Vuforia Engine → Image
  ▶ 마커를 인식해 특정 객체를 활성화하는 역할
     ▶ ImageTarget의 3가지 스크립트
        ▶ ImageTargetBehaviour = 마커의 유형, 데이터베이스, 마커를 선택하는 스크립트
        ▶ TurnOffBehaviour Mesh Renderer와 Mesh Filter를 런타임에서 비활성화하는 스크립트  
        ▶ AR 카메라가 마커를 인식하고 트래킹을 시작하거나 트래킹이 종료됐을 때 이벤트를 발생시키는 스크립트
     ▶ ImageTarget은 런타임 시 DefaultTrackableEventHandle 스크립트에서 AR카메라의 트래킹이 시작되기 전까지 ImageTarget의 하위에 있는 모든 게임오브젝트의 Renderer 계열 컴포넌트와 Collider 컴포넌트를 비활성화한다. 
  
※ Vuforia을 통해 AR상에서 이미지를 뛰우기 위해서는 ImageTarget아래로 원하는 오브젝트를 차일드한다.

● ARCamera
  ▶ GameObject → Vuforia Engine → ARCamera
  ▶ Vuforia용 Camera
     ● Vuforia Behaviour 컴포넌트
       ● World Center Mode
         ▶ DEVICE : Device를 원점으로 생각?
         ▶ FIRST_TARGET : 첫 번째로 인식한 Image Target을 원점으로 설정  

● 트래킹 이벤트
  ▶ ITrackableEventHandle 인터페이스를 통해 트래킹 여부에 따라 특정 로직을 수행하는 함수로 응용할 수 있다.

● Extended Tracking
  ▶ AR 카메라의 트래킹이 끊어졌을 때에도 마커에 있던 오브젝트를 계속 표현하기 위해 사용
  ▶ Project View → Resources → VuforiaConfiguration을 선택한 후 → Device Tracker의 Track Device Pose 옵션에 체크

● 여러 개의 마커 인식
  ▶ Project View → Resources → VuforiaConfiguration → Vuforia 섹션 → Max Simultaneous Tracked Images 속성 변경 (숫자 만큼 마커를 트래킹한다는 의미)

    ▶ Max Simultaneous Tracked Object = 3D Object 마커에 한해 동시에 트래킹할 수 있는 마커의 개수
